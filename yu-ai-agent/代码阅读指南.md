## 一、项目概述

### 1.1 项目简介

**Yu-AI-Agent** 是一个基于 Spring Boot 3.4 和 Spring AI 框架构建的智能代理（AI Agent）系统。项目实现了一个具有自主规划能力的超级智能体（YuManus），并以"恋爱大师"应用作为业务场景示例，展示了 AI Agent 的完整开发流程。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🤖 **ReAct 智能体** | 实现思考-行动（Think-Act）循环的自主规划智能体 |
| 💬 **多轮对话** | 支持基于文件持久化的会话记忆 |
| 🔍 **RAG 知识库** | 集成向量数据库的检索增强生成 |
| 🛠️ **工具调用** | 支持网页搜索、文件操作、PDF生成等多种工具 |
| 🌐 **MCP 协议** | 集成 Model Context Protocol 扩展能力 |
| 📡 **流式输出** | 支持 SSE 实时流式响应 |

### 1.3 技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                      技术栈总览                              │
├─────────────────────────────────────────────────────────────┤
│  框架层：Spring Boot 3.4 + Spring AI 1.0.0-M6              │
│  AI 服务：阿里云通义大模型（DashScope）/ Ollama（可选）       │
```

#### 3.1.2 BaseAgent - 基础智能体

**核心职责**：管理智能体状态和执行流程

```java
// 文件：src/main/java/com/yupi/yuaiagent/agent/BaseAgent.java

public abstract class BaseAgent {
    // 核心属性
    private String name;                    // 智能体名称
    private String systemPrompt;            // 系统提示词
    private String nextStepPrompt;          // 下一步提示词
    private AgentState state = AgentState.IDLE;  // 当前状态
    private int currentStep = 0;            // 当前步骤
    private int maxSteps = 10;              // 最大步骤数
    private ChatClient chatClient;          // AI 客户端
    private List<Message> messageList;      // 消息上下文

    // 核心方法：运行智能体
    public String run(String userPrompt) {
        // 1. 状态校验
        // 2. 执行循环：for (i < maxSteps && state != FINISHED)
        // 3. 调用抽象方法 step()
        // 4. 清理资源
    }
    
    // 支持 SSE 流式输出
    public SseEmitter runStream(String userPrompt) { ... }
    
    // 抽象方法：单步执行（由子类实现）
    public abstract String step();
}
```

**状态机**：

```
  IDLE ──run()──► RUNNING ──完成/超限──► FINISHED
    │                │
    │                │ 异常
    │                ▼
    └─────────────► ERROR
```

#### 3.1.3 ReActAgent - ReAct 模式智能体

**核心职责**：实现 Think-Act 循环模式

```java
// 文件：src/main/java/com/yupi/yuaiagent/agent/ReActAgent.java

public abstract class ReActAgent extends BaseAgent {
    
    // 思考：决定是否需要行动
    public abstract boolean think();
    
    // 行动：执行具体操作
    public abstract String act();
    
    // 实现父类的 step 方法
    @Override
    public String step() {
        boolean shouldAct = think();  // 先思考
        if (!shouldAct) {
            return "思考完成 - 无需行动";
        }
        return act();  // 再行动
    }
}
```

**ReAct 执行流程**：

```
┌─────────────────────────────────────────────────────────────────┐
│                      ReAct 循环流程                              │
│                                                                  │
│    ┌──────────┐      需要行动      ┌──────────┐                 │
│    │  Think   │ ──────────────────► │   Act    │                │
│    │  (思考)   │                     │  (行动)   │                │
│    └────┬─────┘                     └────┬─────┘                │
│         │                                │                       │
│         │ 无需行动                        │                       │
│         │                                │                       │
│         ▼                                ▼                       │
│    ┌──────────────────────────────────────────┐                 │
│    │              返回步骤结果                  │                 │
│    └──────────────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.1.4 ToolCallAgent - 工具调用智能体

**核心职责**：具体实现 think() 和 act() 方法，管理工具调用

```java
// 文件：src/main/java/com/yupi/yuaiagent/agent/ToolCallAgent.java

public class ToolCallAgent extends ReActAgent {
    
    private final ToolCallback[] availableTools;    // 可用工具列表
    private ChatResponse toolCallChatResponse;      // 工具调用响应
    private final ToolCallingManager toolCallingManager;  // 工具调用管理器
    
    @Override
    public boolean think() {
        // 1. 拼接消息上下文
        // 2. 调用 AI 模型获取工具调用建议
        ChatResponse chatResponse = getChatClient().prompt(prompt)
                .system(getSystemPrompt())
                .tools(availableTools)    // 传入可用工具
                .call()
                .chatResponse();
        
        // 3. 解析要调用的工具列表
        List<ToolCall> toolCallList = assistantMessage.getToolCalls();
        
        // 4. 返回是否需要调用工具
        return !toolCallList.isEmpty();
    }
    
    @Override
    public String act() {
        // 1. 执行工具调用
        ToolExecutionResult result = toolCallingManager.executeToolCalls(prompt, toolCallChatResponse);
        
        // 2. 更新消息上下文
        setMessageList(result.conversationHistory());
        
        // 3. 检查是否调用了终止工具
        if (terminateToolCalled) {
            setState(AgentState.FINISHED);
        }
        
        return results;
    }
}
```

### 3.6 Advisor 拦截器

Advisor 是 Spring AI 的核心扩展机制，类似于 AOP 的拦截器。

#### 3.6.1 日志拦截器

```java
// 文件：src/main/java/com/yupi/yuaiagent/advisor/MyLoggerAdvisor.java

public class MyLoggerAdvisor implements CallAroundAdvisor, StreamAroundAdvisor {
    
    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        // 请求前：记录用户输入
        log.info("AI request: {}", request.userText());
        
        // 执行请求
        AdvisedResponse response = chain.nextAroundCall(request);
        
        // 请求后：记录 AI 响应
        log.info("AI response: {}", response.response().getResult().getOutput().getText());
        
        return response;
    }
}
```

```java
// 文件：src/main/java/com/yupi/yuaiagent/advisor/ReReadingAdvisor.java

public class ReReadingAdvisor implements CallAroundAdvisor, StreamAroundAdvisor {
    
    private AdvisedRequest before(AdvisedRequest advisedRequest) {
        // 让 AI 再读一遍用户问题，提高推理能力
        return AdvisedRequest.from(advisedRequest)
                .userText("""
                    {re2_input_query}
                    Read the question again: {re2_input_query}
```

2. 注册工具（`ToolRegistration.java`）：

```java

public ToolCallback[] allTools() {
    return ToolCallbacks.from(
        // ... 其他工具
        new MyCustomTool()
    );
}
```

### 7.2 创建新的智能体

```java
@Component
public class MyAgent extends ToolCallAgent {
    
    public MyAgent(ToolCallback[] tools, ChatModel chatModel) {
        super(tools);
        this.setName("myAgent");
        this.setSystemPrompt("Your system prompt here...");
        this.setNextStepPrompt("Your next step prompt...");
        this.setMaxSteps(15);
        this.setChatClient(ChatClient.builder(chatModel).build());
    }
}
```

### 7.3 添加新的 Advisor

```java
public class MyAdvisor implements CallAroundAdvisor {
    
    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        // 前置处理
        request = preProcess(request);
        
        // 调用下一个 Advisor
        AdvisedResponse response = chain.nextAroundCall(request);
        
        // 后置处理
        return postProcess(response);
    }
    
    @Override
    public int getOrder() {
        return 0;  // 执行顺序
    }
    
    @Override
    public String getName() {
        return "MyAdvisor";
    }
}
```

## 八、关键设计模式

### 8.1 模板方法模式

`BaseAgent` 定义了智能体执行的骨架算法，子类实现具体步骤：

```
BaseAgent.run() ── 定义执行流程骨架
    │
    └── step() ── 抽象方法，由子类实现
           │
           └── ReActAgent.step() ── 实现 Think-Act 循环
                    │
                    ├── think() ── 抽象方法
                    └── act()   ── 抽象方法
                           │
                           └── ToolCallAgent ── 具体实现
```

### 8.2 策略模式

工具调用通过 `ToolCallback` 接口实现策略模式，不同工具实现不同策略。

### 8.3 责任链模式

`Advisor` 拦截器链实现了责任链模式，多个 Advisor 按顺序处理请求。
